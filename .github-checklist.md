# GitHub Public Release Checklist

Dit bestand bevat de checklist voor het publiceren van LapEdge op GitHub. **Verwijder dit bestand voordat je pusht, of voeg toe aan .gitignore.**

## ‚úÖ Security & Privacy Checklist

### Environment Variables
- [x] Geen .env bestanden in repository
- [x] .env.example is aanwezig (zonder echte credentials)
- [x] .gitignore bevat alle .env varianten
- [x] Alle environment variabelen gebruiken `process.env.VAR` (niet hardcoded)

### API Keys & Credentials
- [x] Geen API keys in code
- [x] Geen passwords in code
- [x] Geen tokens in code
- [x] Geen private keys in repository
- [x] Alle externe URLs zijn publiek toegankelijk (geen auth parameters)

### Source Code
- [x] Geen persoonlijke informatie (emails, telefoonnummers, adressen)
- [x] Geen interne bedrijfsinformatie
- [x] Geen comments met gevoelige info (TODO's met passwords etc.)
- [x] Geen hardcoded database credentials
- [x] Geen AWS/Azure/GCP credentials

### Dependencies
- [x] Geen private npm packages in package.json
- [x] Alle dependencies zijn publiek beschikbaar
- [x] package-lock.json kan veilig gecommit worden
- [x] Geen node_modules in git (staat in .gitignore)

### Files & Directories
- [x] IDE configuratie niet in git (.vscode, .idea, etc.)
- [x] OS specifieke bestanden niet in git (.DS_Store, Thumbs.db)
- [x] Build directories niet in git (dist/, build/)
- [x] Log bestanden niet in git
- [x] Temporary files niet in git

## üì¶ Repository Setup

### Voor eerste push naar GitHub:

```bash
# 1. Controleer wat er gecommit wordt
git status
git diff

# 2. Voeg nieuwe files toe (selectief!)
git add ARCHITECTURE.md
git add CHANGELOG.md
git add CONTRIBUTING.md
git add README.md
git add .gitignore

# 3. Commit wijzigingen
git commit -m "docs: add comprehensive documentation for public release"

# 4. Controleer dat er geen gevoelige bestanden zijn
git ls-files | grep -E "(\.env$|secret|credential|\.key$|\.pem$)"
# Dit moet LEEG zijn!

# 5. Create GitHub repo (via GitHub web interface)
# Maak een nieuwe PUBLIC repository op github.com

# 6. Add remote en push
git remote add origin https://github.com/jouw-username/lapedge.git
git branch -M main
git push -u origin main
```

### Na de push:

1. **GitHub Settings configureren:**
   - [ ] Repository description toevoegen
   - [ ] Topics toevoegen: `speed-skating`, `react`, `nodejs`, `real-time`, `sports`
   - [ ] Website URL toevoegen (als je deploy)
   - [ ] Issues inschakelen
   - [ ] Wiki uitschakelen (tenzij je die gebruikt)
   - [ ] Discussions overwegen (voor community)

2. **README badges toevoegen (optioneel):**
   ```markdown
   ![License](https://img.shields.io/github/license/username/lapedge)
   ![Last Commit](https://img.shields.io/github/last-commit/username/lapedge)
   ![Issues](https://img.shields.io/github/issues/username/lapedge)
   ```

3. **GitHub Actions overwegen:**
   - CI/CD pipeline voor automated testing
   - Automatic dependency updates (Dependabot)
   - Code quality checks

## üö® LAATSTE CONTROLE voor Push

Run deze commands:

```bash
# Zoek naar potenti√´le secrets
git grep -i "password"
git grep -i "api.key"
git grep -i "secret"
git grep -i "token"
git grep -E "[0-9]{4,}"  # Credit card nummers etc

# Controleer .env files
ls -la | grep env

# Controleer tracked files
git ls-files | grep -E "(secret|credential|\.key|\.pem|\.env$)"
```

Als een van deze commands iets vindt dat gevoelig is: **NIET PUSHEN!**

## üéØ Post-Release

Na publicatie op GitHub:

1. **Monitor voor secrets:**
   - GitHub zal automatisch scannen op leaked secrets
   - Let op security alerts in repository settings

2. **Community setup:**
   - Add CODEOWNERS file (optioneel)
   - Setup issue templates
   - Setup PR templates
   - Add branch protection rules

3. **Documentatie links:**
   - Update README met GitHub specifieke links
   - Add "Star" en "Fork" badges
   - Link naar Issues voor bug reports

## ‚ö†Ô∏è Als je per ongeluk een secret hebt gecommit:

**NIET alleen de file verwijderen in een nieuwe commit!**

Git history bevat nog steeds het secret. Je moet:

1. **Immediately rotate de secret** (nieuwe API key aanmaken, oude invalideren)
2. **Remove from git history:**
   ```bash
   # Voor recente commit:
   git reset --soft HEAD~1

   # Voor oudere commits:
   git filter-branch --force --index-filter \
     "git rm --cached --ignore-unmatch PATH/TO/SECRET/FILE" \
     --prune-empty --tag-name-filter cat -- --all

   # Of gebruik BFG Repo-Cleaner
   ```
3. **Force push** (als repo al public is, waarschuw users!)
